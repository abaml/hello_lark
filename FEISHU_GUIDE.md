# 飞书与 Cody-Bot 交互指南

## 快速开始
在飞书聊天窗口中直接发送消息即可与 Cody-Bot 交互，支持以下场景：

### 1. 自然语言对话
直接提问或下达指令：
```
帮我分析这个项目的依赖结构
请创建一个新的飞书文档
解释一下这段代码的逻辑
```

### 2. 执行系统命令
使用 `!` 前缀执行终端命令：
```
! ls -l
! git status
! npm install
```

### 3. 文件操作
直接发送文件路径进行读写：
```
读取 package.json 文件
修改 src/index.js 中的第10行
创建新文件 docs/README.md
```

### 4. 飞书文档操作
- **创建文档**：`创建飞书文档，标题为"项目计划"，内容为"## 目标\n- 完成MVP开发"`
- **更新文档**：`更新文档 https://example.feishu.cn/docx/xxx，在末尾添加"## 新章节"`
- **查询文档**：`获取文档 https://example.feishu.cn/docx/xxx 的内容`

## 特殊场景处理

### 长内容处理
对于深度分析报告等长内容，Cody-Bot 会自动创建飞书文档并发送链接：
```
帮我分析项目架构并生成详细报告
```

### 危险操作确认
执行以下操作前需要二次确认：
- `rm -rf`、文件删除
- `git push --force`、`git reset --hard`
- 其他不可逆的批量修改

格式示例：
```
我将执行 rm -rf dist/，回复'确认'继续
```

## 高级功能

### 1. 多步骤任务
支持连续对话完成复杂任务：
```
帮我启动开发服务器
> 服务器已启动在端口3000
帮我打开浏览器访问 http://localhost:3000
```

### 2. 上下文保持
Cody-Bot 会自动保留对话上下文，无需重复说明背景信息：
```
这个项目用什么框架开发的？
> 使用 React + Node.js
帮我添加一个新的路由
> 已在 src/routes.js 中添加路由配置
```

### 3. 错误处理
执行失败时会自动提示并提供解决方案：
```
! npm install
> 安装失败：缺少 package.json
帮我初始化 npm 项目
```

## 注意事项
1. 确保本地 Cody-Bot 服务正在运行
2. 飞书消息会实时同步到本地终端
3. 敏感操作请谨慎确认，避免数据丢失
4. 大文件操作建议通过飞书文档链接分享，而非直接上传

## 问题反馈
如果遇到交互问题，请在终端执行：
```
! cody-bot --debug
```
查看详细日志或联系管理员。
